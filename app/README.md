# NewtID — демо (Streamlit), браузерное опознание тритона

Браузерное MVP-демо re-ID тритонов по узору брюшка. Запускается **локально, offline**
(целевой сценарий — ноутбук заказчика без интернета). Концепция как у существующих систем фотоидентификации:
загрузка/выбор фото → top-K похожих особей карточками с уверенностью → решение known/new.
**Дифференциатор:** side-by-side оверлей совпавших пятен (матчинг созвездия центроидов — embedding-only системы на DINOv2 этого не дают).

## Запуск
```bash
cd "<корень репозитория>"
pip install -e ".[crop]"      # ядро (numpy/torch/timm/opencv/…)
pip install streamlit         # UI
streamlit run app/demo.py     # → откроется http://localhost:8501
```
Откроется в браузере. Сервер локальный, интернет не нужен (кроме ПЕРВОЙ загрузки весов MegaDescriptor с HF —
дальше offline; для режима «Выбрать пробу» модель вообще не требуется).

## Что нужно на диске
**В репозитории:** код демо (`app/demo.py`), бэкенд (`triton_crop.demo_backend`) и сводка
sealed-метрик `artifacts/ab_test_headline.json` (вкладка «О системе»; без этого файла демо
не падает — показывает только dev-числа).

**Получить отдельно (в git не попадает):**
- `artifacts/runs/belly_{seg,pose}/weights/best.pt` — веса YOLO (нужны только для пути
  «загрузить сырое фото»); после публикации — скачать из GitHub Releases / HF Hub
  (ссылка добавится при публикации).
- `artifacts/embed/oof/` (предвычисленные эмбеддинги галереи (OOF) + md5/role/individual_id/cohort)
  и `crops_belly/<md5>.png` (кропы брюшка: галерея + dev-пробы) — производные данных заказчика,
  не распространяются. Без них недоступны режим «Выбрать пробу» и вкладка «Особи»;
  остаётся путь «Загрузить фото» (при наличии весов).

## Режимы (вкладка «Опознание»)
- **Выбрать пробу (dev, надёжно)** — берётся известная dev-особь (эмбеддинг предвычислен) → мгновенно, БЕЗ загрузки
  модели. Рекомендуется для живого демо на защите.
- **Загрузить фото** — своё фото: сегментация (YOLO) → кроп → эмбеддинг (MegaDescriptor, ленивая загрузка).
  Можно отметить «это уже кроп брюшка», чтобы пропустить сегментацию.

Ещё вкладки: **«Особи»** (просмотр базы) и **«О системе»** (архитектура + честные метрики + оговорки калибровки).

## Честность (для защиты)
Шкала «уверенность» — калиброванная **монотонная** (не вероятность); на близких особях (PW) возможна сатурация —
это ограничение open-set (AUROC≈0.45 sealed-test / ≈0.58 dev), отражено в выводах ВКР. Финальные числа ВКР
показываются во вкладке «О системе» (читаются из `artifacts/ab_test_headline.json`). Логика опознания (`triton_crop.demo_backend`)
покрыта тестами (`tests/test_demo_backend.py`); рендер приложения проверяется headless-прогоном `streamlit run app/demo.py` (HTTP 200, без ошибок старта).
